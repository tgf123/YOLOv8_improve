# RT-DETR/DEIM 模型改进模块库指南

## 一、项目介绍
本模块库收集了 **206个 可插拔深度学习模块**，专门面向 RT-DETR、DEIM 等单/双模态目标检测模型的迭代优化需求，覆盖自注意力增强、卷积优化、特征融合、频域处理、骨干网络等12个核心技术方向。所有模块均经过适配，可直接替换原模型对应组件，针对**小目标检测差、遮挡漏检、低质图像性能下降、多尺度目标适配难**等痛点场景优化，帮助开发者快速完成模型选型与改进。

---

## 二、模块分类说明

为方便快速匹配改进需求，将所有模块分为以下12类，每类标注了适配的RT-DETR/DEIM替换场景：

| 序号 | 分类 | 模块数 | 技术特点 | 适配替换场景 |
| --- | --- | --- | --- | --- |
| 1 | 自注意力机制 | 67 | 各类自注意力变体（含线性注意力），基于Q·Kᵀ·V或其近似计算全局像素级依赖，BNC格式输入输出。 | 替换RT-DETR的AIFI模块、HybridEncoder自注意力层 |
| 2 | 通用注意力机制 | 22 | 通道注意力、空间注意力、混合注意力等非QKV自注意力类模块，BCHW格式输入输出，无全局O(N²)计算开销，对特征进行加权增强。 | 替换Backbone/Neck中的SE、CA等注意力层 |
| 3 | Mamba类模块 | 11 | 基于Mamba/SSM架构的序列建模模块，需导入mamba_ssm库，线性复杂度O(N)捕获长程依赖。 | 替换AIFI模块、HybridEncoder的Encoder层 |

| 11 | 归一化/激活优化 | 6 | 改进版归一化、激活函数，提升训练稳定性、收敛速度。 | 替换原模型的BN、ReLU等基础组件 |

### 12. 骨干网络（11个）


完整骨干网络模块，均实现了DEIM统一Wrapper接口（`*Backbone`类），对齐HGNetV2接口规范，支持YAML配置注入。直接替换RT-DETR/DEIM的Backbone，输出多尺度特征图。


**统一接口**：所有骨干Wrapper类的`__init__`签名一致：
```python
XxxBackbone(
    name='variant_name',    # 模型变体名称
    pretrained=False,       # 是否加载预训练权重
    return_idx=[2, 3],     # 返回哪些stage的特征（2=stride8, 3=stride16, 4=stride32）
    freeze_at=-1,           # 冻结前N个stage（-1=不冻结）
    freeze_norm=False,      # 是否冻结归一化层
    local_model_dir='',     # 本地预训练权重路径
)
```
**统一输出**：`forward(x)` 返回 `list[Tensor]`，每个Tensor形状为 `(B, C, H, W)`，由`return_idx`控制输出哪些stage。

- **其他骨干**：仅需`torch`+`timm`，无额外依赖

  type: MambaOutBackbone
  name: mambaout_tiny
  return_idx: [2, 3]
  pretrained: False
  freeze_at: -1

## 四、改进选型建议
1. **小目标检测优化**：优先选择`BIE`、`DEConv`、`EGA`、`LEGM`等边缘/小目标增强模块，或`LSKA`、`LSConv`等大核卷积模块，提升小目标特征提取能力。
2. **遮挡场景优化**：优先选择`Defensive_Refinement_Module`、`Mask_Attention`等遮挡修复/掩码注意力模块，或`DSAM`、`DSSA`等稀疏注意力模块，降低遮挡特征干扰。
3. **低质/夜间/模糊图像优化**：优先选择`AdaFD`、`FBGW`、`FFCM`等频域处理模块，或`EVSSM`、`MambaLayer`等Mamba类模块，提升抗噪声和纹理提取能力。
4. **计算量优化**：优先选择`Agent_attention`、`Biformer`、`SSA`等线性/稀疏自注意力模块，或`Repconv`、`LSKA`等轻量化卷积模块，降低模型计算量。
5. **双模态（RGB+IR）优化**：优先选择`DAAFM`、`CSI`、`FAMamba_CSI`等双模态融合模块，提升多模态特征交互能力。

## 五、使用说明
1. 直接将目标模块的代码替换RT-DETR/DEIM原模型中对应组件，保持输入输出维度一致即可。
2. 部分模块支持自定义参数（如通道数、注意力头数、卷积核大小等），可根据实际任务调整。
3. 改进时可优先替换`AIFI`模块、Backbone/Neck的注意力/卷积层，这些位置对性能提升最明显。
4. **骨干网络替换**：直接在YAML配置中将`type: HGNetV2`替换为`type: XxxBackbone`，并设置对应的`name`参数选择变体。所有骨干网络均对齐HGNetV2接口，无需修改下游代码。
5. **自注意力模块输入输出格式说明**：所有(B,N,C)→(B,N,C)格式的模块（自注意力机制类），forward的输入输出统一为**BNC格式**（`x.shape = [B, N, C]`，N=H*W），而非BCHW格式。模块内部会自动完成BNC↔BCHW维度变换，调用方无需额外处理。具体变换逻辑：
   - **BNC→BCHW**（forward开头）：`B, N, C = x.shape; H = W = int(N**0.5); x = x.transpose(1, 2).reshape(B, C, H, W)`
   - **BCHW→BNC**（forward结尾）：`B, C, H, W = x.shape; x = x.flatten(2).transpose(1, 2)`
   - 注意：H=W要求空间维度为方形，即H=W，N=H*W须为完全平方数。
6. **Mamba类模块依赖说明**：本类所有模块均需导入`mamba_ssm`库（`pip install mamba-ssm`），依赖CUDA编译环境。若无法安装，可选择同维卷积+注意力类模块作为替代。
7. **双输入特征融合模块说明**：以下15个模块支持双输入特征融合，已适配RT-DETR双模态场景：
   - **不同空间尺度输入**（自动上采样对齐到大尺寸）：DPCF、DPCF_v2、CSFblock、HAFB、MASAG、CMA、CVIM、DFF、EFC、FEFM、HFFE、mdfm、MFM
     - 输入：`(B,C1,H1,W1) + (B,C2,H2,W2)`，H1/H2、W1/W2可不同
     - 输出：`(B,out_C,max_H,max_W)`，自动上采样小尺寸到大尺寸
     - 示例：`[2,128,40,40] + [2,64,20,20] → [2,256,40,40]`
     - ⚠️ DPCF_v2已知bug：输出空间尺寸为小尺寸而非大尺寸（输出20×20而非40×40），待修复
   - **相同空间尺度输入**：DAAFM(BNC格式)、FFAFusion
     - 输入：`(B,C1,H,W) + (B,C2,H,W)`，空间尺寸相同，通道可不同
     - 输出：`(B,out_C,H,W)`，输出通道可配置
     - 示例：`[2,128,40,40] + [2,128,40,40] → [2,256,40,40]`
   - **初始化参数统一模式**：`Module(c1=输入1通道, c2=输入2通道, out_channels/out_d/out_features=输出通道)`
   - **空间对齐方式**：统一使用`F.interpolate(mode='bilinear', align_corners=False)`
   - **输出通道控制**：统一使用1x1 Conv2d投影
